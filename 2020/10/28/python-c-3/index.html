本篇主要尝试对各类网站进行爬取练习。

京东某商品信息

例如：https://item.jd.com/100008667323.html
import requests
kv = {'user-agent':'Mozilla/5.0'}	#浏览器身份标识
url = "https://item.jd.com/100008667323.html"
try:
    r = requests.get(url,headers=kv)
    r.raise_for_status()
    r.encoding = r.apparent_encoding
    print(r.text)
except:
    pritnf("爬取失败")

注：京东提供了页面编码utf-8，r.encoding与r.apparent_encoding一致。

如果不加user-agent，r.status_code依然为200，但返回结果为登录页面。

因为京东对爬虫有一定的限制，如果程序中不写明user-agent，访问者默认为python的一个程序，从而访问出错。解决办法只需将程序模拟为浏览器Mozilla/5.0，即可正常访问。

 

亚马逊某商品信息

例如：https://www.amazon.cn/dp/B01HIUT4D8
import requests
kv = {'user-agent':'Mozilla/5.0'}
url = "https://www.amazon.cn/dp/B01HIUT4D8"
cookie = {
}
try:
    r = requests.get(url,cookies=cookie,headers=kv)
    r.raise_for_status()
    r.encoding = r.apparent_encoding
    print(r.text)
except:
    pritnf("爬取失败")

与京东略有不同，需要将cookie写入，否则回应信息会显示被识别为机器人。

cookie即某些网站为了辨别用户身份，进行Session跟踪而储存在用户本地终端上的数据（通常经过加密）。

 

百度360搜索关键词提交

百度的关键词接口：http://www.baidu.com/s?wd=keyword

360的关键词接口：http://www.baidu.com/s?q=keyword

以百度为例，360一致。

import requests
kv = {'wd':'python'}
url = "http://www.baidu.com/s"
try:
    r = requests.get(url,params=kv)
    r.raise_for_status()
    r.encoding = r.apparent_encoding
    print(r.request.url)
    print(len(r.text))
except:
    pritnf("爬取失败")

第一个输出的是get获取的url链接，可以看到键值对已经附加到了原url之后。

第二个输出的是获取内容长度，内容过多就没有进行输出。（两年前还是30w，现在近60w）

 

网络图片的爬取和存储

网络图片链接的格式：

http://www.example.com/picture.jpg

国家地理：

http://www.nationalgeographic.com.cn

以国家地理为例，选择一个图片web页面：

http://www.ngchina.com.cn/photocontest2020/images/top.jpg

代码如下

import requests
path = "D:/abc.jpg"
url = "http://www.ngchina.com.cn/photocontest2020/images/top.jpg"
r = requests.get(url)
with open(path, 'wb') as f:
    f.write(r.content)  #r.content为返回内容的二进制形式
f.close()

 

但这种程序是不完善的，还需要考虑各种情况。

简单进行修饰一下

import requests
import os
root = "D://pics//"
url = "http://www.ngchina.com.cn/photocontest2020/images/top.jpg"
path = root + url.split('/')[-1]
try:
    if not os.path.exists(root):
        os.mkdir(root)
    if not os.path.exists(path):
        r = requests.get(url)
        with open(path, 'wb') as f:
            f.write(r.content)
            f.close()
            print("文件保存成功")
    else:
        print("文件已存在")
except:
    pritnf("爬取失败")

 

IP地址归属地的自动查询

python没有关于ip归属地的库文件。我们可以通过ip138进行查询。

https://www.ip138.com/

通过搜索任意一个ip地址可以得知，该url形式为

https://www.ip138.com/iplookup.asp?ip=ipaddress&action=2

import requests
kv = {'user-agent':'Mozilla/5.0'}
url = "https://www.ip138.com/iplookup.asp?ip="
try:
    r = requests.get(url + 'ip地址' + '&action=2',headers=kv)
    r.raise_for_status()
    r.encoding = r.apparent_encoding
    print(r.text[1000:2000])
except:
    pritnf("爬取失败")

尽管该网站的robots协议备注允许所有爬虫爬取，但实际运行过程中，不伪装成浏览器是无法进行正常访问的。
